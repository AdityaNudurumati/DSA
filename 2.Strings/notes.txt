--------------------------------------------------
How this section is organized
--------------------------------------------------
Each numbered folder is one pattern with a notes.txt (sub-pattern tree, template,
and a вҳҗ/вҳ‘ problems checklist). When a pattern is "built out", it is split into
sub-folders (one per sub-pattern), each holding one runnable .py per problem.

Trigger phrases (identify the pattern before coding):
- Sorted/compare ends            -> Two Pointers (opposite ends)
- Contiguous substring + window  -> Sliding Window
- Counts / anagrams / dedup      -> Hashing / Frequency
- "find pattern in text"         -> Pattern Matching (KMP / Rabin-Karp / Z)
- "palindrome"                   -> Palindrome Techniques
- Build/format/parse output      -> String Construction
- Prefix / dictionary / autocomplete -> Trie
- Balanced / nested / decode     -> Stack
- Rearrange / lexicographically smallest -> Greedy
- Two strings, optimal transform -> Dynamic Programming
- Many substring comparisons     -> Rolling Hash
