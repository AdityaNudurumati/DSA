--------------------------------------------------
Three ideas behind almost every queue question
--------------------------------------------------
Queue            -> BFS and processing in arrival order
Deque            -> sliding windows and specialized BFS (0-1 BFS)
Priority Queue   -> ordering by priority instead of arrival time

--------------------------------------------------
Suggested interview order
--------------------------------------------------
1 Basic FIFO → 7 Design → 2 BFS/Level Order → 3 Circular → 5 Deque →
4 Monotonic Deque → 6 Scheduling → 9 (Multi-Source → State BFS → 0-1 BFS → Bidirectional)

Must-solve: Implement Queue, Number of Recent Calls, Queue using Stacks (easy);
Binary Tree Level Order, Rotting Oranges, Design Circular Queue/Deque,
Sliding Window Maximum, Dota2 Senate (medium); Word Ladder, Shortest Subarray with
Sum at Least K, Jump Game VI, Open the Lock (hard).

--------------------------------------------------
How this section is organized
--------------------------------------------------
Each numbered folder is one pattern with a notes.txt (sub-pattern tree, template,
☐/☑ checklist). When built out, a pattern is split into sub-folders, each with one
runnable .py per problem. BFS files define their own minimal TreeNode / adjacency /
grid inputs so each stays self-contained and runnable.

Core idea
--------------------------------------------------
FIFO. Python: collections.deque — append()/popleft() for a queue,
appendleft()/pop() for the other end. heapq for priority queues.
