==================================================
MATRIX — 3. SIMULATION
==================================================

No clever algorithm — just step through the rules carefully. The recurring trick is
doing it IN-PLACE: encode extra state into existing cells (markers / bit flags) so
you don't allocate a second matrix.

When to think of this?
- "apply these state-transition rules to the grid"
- in-place updates where naive copying would cost O(m*n) extra space

--------------------------------------------------
Tricks
--------------------------------------------------
- Markers: reuse row 0 / col 0 to flag which rows/cols to zero (Set Matrix Zeroes).
- Bit-encoding: store next state in a higher bit so the current pass still reads the
  original via "& 1"; shift at the end (Game of Life).
- Directions: dirs = [(-1,0),(1,0),(0,-1),(0,1)]  (+ diagonals for 8-neighbor).
