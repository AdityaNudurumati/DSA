| 3Sum                 | Sort + Two Pointers   | O(n²) | O(1)  |


A useful interview habit is: first identify the pattern before writing code. For these problems, the trigger phrases are:

Sorted array → Think Two Pointers.
In-place modification → Think Slow/Fast Pointer.
Compare both ends → Think Opposite Pointers.
Need triplets/pairs in sorted data → Sort + Two Pointers.
Partition/group elements → Dutch National Flag / Partition pattern.

1. Opposite Ends (Left → Right ←)

Two pointers start at opposite ends and move toward each other.

When to think of this?
Sorted array
Pair sum problems
Palindrome checking
Maximizing/minimizing something between two ends

Problems:
    Two Sum II
    Container With Most Water
    Valid Palindrome
    Squares of a Sorted Array
    3Sum (inner loop)
    4Sum (inner loop)
    Trapping Rain Water
    Boats to Save People

Template:
left = 0
right = len(nums) - 1

while left < right:

    if condition:
        left += 1

    else:
        right -= 1

2. Same Direction (Slow/Fast)

Both pointers move from left to right.

When to think of this?
In-place array modification
Remove duplicates
Move elements
Compress data

Problems
    Remove Duplicates from Sorted Array
    Move Zeroes
    Remove Element
    Sort Array By Parity
    Merge Sorted Array

Template:
slow = 0

for fast in range(len(nums)):

    if condition:
        nums[slow] = nums[fast]
        slow += 1

3. Sliding Window (Special Same Direction)

Technically a Two Pointer pattern, but promoted to its own pattern in this
roadmap. Full notes, templates, and problems live in ../2.sliding_window/.
The summary below is kept just to show the relationship.

Both pointers move forward, maintaining a window.

Problems
    Maximum Sum Subarray of Size K
    Longest Substring Without Repeating Characters
    Minimum Window Substring
    Fruits Into Baskets

Template:
left = 0

for right in range(len(nums)):

    while invalid_window:
        left += 1
